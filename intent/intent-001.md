# intent-001 — Interactive product-direction capability

## Verbatim anchors

2026-07-09: "it feels like I'm missing a product manager. There is a
product owner that keeps after the PDRs, but there isn't really a role
or activity that is brainstorming with a product direction outcome as
opposed to a specific incremental feature implementation."

2026-07-09: "There is also a lot of things that seem to fall back on
the router since it is the only interactive role, but it is often
acting like far more than a simple router."

2026-07-09: "I want people who use this system to be able to work as
stakeholders that are supported by these different roles."

2026-07-09 (scope): "I'm currently trying to create a system for
people who have an idea of what they want to build and want to use
spec-driven methods to accomplish that. I'm not currently trying to
create a full system for taking a product to market."

2026-07-09 (scope extension): "while not a full traditional go to
market, the PM role can work with and maintain the current state doc
and it can communicate the product value in the form of the github
landing README and eventually a static market site."

## The goal behind the ask

Stakeholders of shopsystem-based products should have an interactive,
direction-level product partner — divergent brainstorming that
terminates in durable artifacts — without the router accreting that
work and without abandoning the system's spec-driven discipline.

## Who it serves

Product authorities / stakeholders of shopsystem-based products
(including this bootstrap instance); secondarily the PO and Architect
roles, which receive cleaner inputs.

## Constraints

Router must stay a thin classifier. Subagent topology (PDR-002) is
batch-mode; interactive work must live in the main session. Existing
PO contract work (briefs → scenarios) must not be disrupted.

## Non-goals

Full go-to-market: no market/competitive research, personas,
segmentation, positioning, pricing, growth metrics. (Narrows
PDR-012's 2026-06-05 product-general steering; see pdr-P02.)

## Appetite signal

Not stated numerically; stakeholder accepted the shaped design as-is
("I'm inclined to accept it as-is") and requested drafting. Appetite
for implementation to be set at cand-P01.

## Failure conditions

Interactive sessions that produce nothing durable ("idle chat").
A PM role that duplicates or destabilizes the PO's commitment work.
Router prompt growth resuming.

## Open threads

Whether the PM name imports unwanted GTM connotation; stakeholder
comfortable with PM label scoped by charter.
