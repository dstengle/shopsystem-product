# sess-2026-07-14-b — PM system self-observations

## Outcome

Stakeholder reviewed the PM system's first substantive use
(`sess-2026-07-14-a`, `cand-002`) and raised five direct observations.
Router worked through each rather than defaulting to "capture
everything as an intent":

1. **Candidates lack the `Verbatim anchors` field intent records have**
   — confirmed as a clear, unambiguous typedef gap with no solution-
   space ambiguity. Per the PM-mode entry test itself
   ("a committed contract input... routes to lead-po... only
   direction-shaped input is a PM-mode entry"), this does NOT warrant
   an intent record — it routes directly to lead-po as a committed-
   contract brief targeting shopsystem-knowledge (artifact-shape
   owner), skipping discovery/shaping entirely. Dispatched in this
   session (see below), not captured as an intent.

2. **Probe formalization** — stakeholder has an external handoff coming;
   no action here, noted for later.

3. **Progressive disclosure isn't preventing unscoped retrieval** —
   validated directly against this session's own worst mistake (the
   `findings/fabro-spike/*` citation error in `cand-002`'s shaping).
   Genuine solution-space ambiguity (new tool? tighter grants? a
   retrieval-time gate?) — captured as `intent-004`.

4. **Memory played a role it shouldn't have.** Router had logged a
   durable behavioral correction (verify subagent citations) as a
   personal memory entry rather than a prompt-level fix — stakeholder
   correctly identified this doesn't touch the actual subagent prompts
   (shopsystem-templates-owned, poured, not hand-edited here) and
   suggested prompt adjustment plus possible mechanical checks. Folded
   into `intent-004` alongside #3, since both are the same underlying
   problem (evidence-gathering reliability) from different angles.

5. **Candidate/intent numbering and backlog ordering.** Router's first
   answer conflated the `bd` operational backlog (154 ready issues,
   already has a working priority/dependency mechanism, never touches
   `intent/`) with the PM/product backlog (currently just `intent-003`)
   — stakeholder caught this was the wrong scale/category, not just an
   unclear explanation. Corrected framing captured as `intent-005`,
   scoped narrowly to intent-record ordering only.
