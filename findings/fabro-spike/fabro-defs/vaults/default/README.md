# fabro vault scaffold — DUMMY PLACEHOLDERS ONLY

`secrets.json` is fabro's tier-2 workflow-visible vault
(`<FABRO_STORAGE_DIR>/vaults/default/secrets.json`, managed by
`fabro secret set|list|rm`; format pinned in `../../../00a-fabro-tool.md` §4.2).

**Every value here is `__PLACEHOLDER__` on purpose. Real credentials NEVER live in
fabro.** This satisfies the epic's hard invariant: credentials via **agent-vault, NOT
fabro secrets**.

How real creds actually reach a node at run time (PROVEN for GitHub in `00a` §4.3):

1. fabro resolves the exact-name vault token (e.g. `GITHUB_TOKEN`,
   `ANTHROPIC_API_KEY`) and injects the **placeholder** string into the node's
   execution env — the client only needs *a* non-empty header value.
2. Under `[environments.local] provider='local'` the node inherits the parent env,
   including **`HTTPS_PROXY` → agent-vault**.
3. The agent-vault egress proxy **substitutes the real credential on the wire**
   (headers/path/body/WS frames). fabro holds only the dummy.

The real store is generated by `fabro install` / `fabro secret set`; this file is a
scaffold showing the shape + the placeholder discipline. Regenerating it (e.g.
`fabro secret set GITHUB_TOKEN __PLACEHOLDER__`) is safe — keep values dummy.

**OPEN (Slice 4 / AC6 / AC9):** that a *non-dry-run* agent node's own LLM/tool calls
(not just fabro's GitHub ops) inherit `HTTPS_PROXY` and ride the proxy is the single
sharpest unproven risk. `provider='local'` should inherit for free; confirm live.
